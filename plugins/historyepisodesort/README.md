# 历史记录排序插件 - Vue联邦模块版本

## 概述

本插件已升级为使用Vue联邦模块架构，提供更现代化的前端体验和更好的性能。对同一部电视剧的不同剧集按正确的时间顺序重新排序。

## 功能特性

- 🔄 **自动排序**：按剧集编号重新排序整理时间
- 📺 **电视剧管理**：查看所有电视剧的历史记录
- 🎯 **选择性排序**：支持选中特定电视剧进行排序
- ⚡ **即时执行**：立即运行一次全部排序
- 🎨 **现代UI**：基于Vue 3 + Vuetify 3的响应式界面
- 🔍 **搜索功能**：快速查找特定电视剧

## 技术架构

### 后端
- 基于MoviePilot插件框架
- 使用联邦模块渲染模式：`get_render_mode() -> ("federation", "/assets/remoteEntry.js")`
- 后台调度任务支持
- RESTful API接口设计

### 前端
- **框架**：Vue 3 + Vuetify 3
- **构建工具**：Vite + Module Federation
- **样式**：Material Design Icons
- **HTTP客户端**：Axios
- **数据表格**：Vuetify DataTable

## 开发指南

### 环境要求
- Node.js 16+
- npm 或 yarn

### 开发流程

1. **安装依赖**
   ```bash
   cd web
   npm install
   ```

2. **开发模式**
   ```bash
   npm run dev
   ```
   访问 http://localhost:3002

3. **构建生产版本**
   ```bash
   npm run build
   # 或使用构建脚本
   ./build.sh
   ```

### API接口

- `GET /plugin/HistoryEpisodeSort/tv_histories` - 获取电视剧历史记录
- `POST /plugin/HistoryEpisodeSort/sort_selected` - 排序选中的电视剧
- `POST /plugin/HistoryEpisodeSort/update_time` - 更新剧集时间
- `POST /plugin/HistoryEpisodeSort/run_once` - 立即运行一次

## 配置选项

- **启用剧集排序**：开启/关闭自动排序功能
- **仅处理电视剧**：是否只处理电视剧类型的媒体

## 使用说明

### 主要功能

1. **查看电视剧列表**
   - 显示所有有历史记录的电视剧
   - 包含海报、标题、剧集数、季数等信息
   - 支持搜索功能快速定位

2. **排序操作**
   - **立即运行一次**：对所有电视剧执行排序
   - **选择性排序**：勾选特定电视剧后批量排序
   - **单个排序**：点击单个电视剧的排序按钮

3. **详情查看**
   - 点击眼睛图标查看电视剧详细信息
   - 包含海报、TMDB ID、剧集统计等

### 排序原理

插件会分析同一部电视剧的不同剧集，按照以下规则重新排序：

1. 按季数（Season）排序
2. 同季内按剧集编号（Episode）排序
3. 重新分配整理时间，确保时间顺序正确

## 注意事项

⚠️ **重要提醒**：该操作会修改历史记录的整理时间，建议在执行前备份数据库。

### 安全建议

1. 在生产环境使用前，先在测试环境验证
2. 定期备份MoviePilot数据库
3. 建议在低峰时段执行大批量排序

## 部署注意事项

1. 确保 `dist/assets/remoteEntry.js` 文件存在
2. 静态资源路径正确配置
3. 服务器支持联邦模块加载
4. 确认调度器权限正常

## 故障排除

### 常见问题

1. **页面无法加载**
   - 检查 `remoteEntry.js` 是否存在
   - 确认构建是否成功完成

2. **排序不生效**
   - 检查插件是否启用
   - 确认数据库连接正常
   - 查看日志文件排查错误

3. **数据显示异常**
   - 检查API接口响应
   - 确认数据格式正确

### 重新构建

如果遇到问题，可以重新构建：

```bash
./build.sh
```
